# Техническое задание для Jules: Создание SEO-оптимизированного многостраничного интерфейса EduTech LMS в `edu_tech/front-end/src/app/[landing]/`

**Контекст проекта:** Интеграция публичных страниц в существующий фронтенд на Next.js 16 (App Router) с использованием Tailwind CSS v4 и TypeScript. Проект уже поддерживает кастомную интернационализацию (папка `app/[landing]/`).

---

## 1. Цель задачи

Разработать и сверстать набор адаптивных, интерактивных публичных страниц (Лендинг, Тарифы, Функционал, Контакты) с глубокой SEO-оптимизацией под рынок Казахстана (таргетинг на города: Алматы, Астана, Шымкент, Караганда) для трех локалей (RU, KZ, EN).

---

## 2. Визуальный стиль и дизайн-система (В духе Stripe.com)

Интерфейс должен выглядеть премиально, минималистично и технологично. При верстке на Tailwind CSS v4 строго придерживайся следующих правил:

* **Фоны и Градиенты:** Используй мягкие интерактивные фоны. В Hero-секциях реализуй фирменные размытые подложки Stripe с помощью комбинации градиентов и эффектов размытия (`bg-gradient-to-tr`, `blur-3xl`, `opacity-40`). Цветовые акценты: глубокий индиго, фиолетовый, благородный синий и яркий циан.
* **Типографика:** Крупные, уверенные заголовки `<h1>` и `<h2>` с использованием градиентного текста (`bg-clip-text text-transparent bg-gradient-to-r`). Текст должен быть максимально четким, с увеличенным межстрочным интервалом (`leading-relaxed`) для лучшей читаемости.
* **Карточки и Границы:** Вместо тяжелых теней используй ультра-тонкие полупрозрачные границы (`border-slate-100` или `border-slate-200/60`). Для карточек преимуществ примени концепцию модульной сетки (Bento Grid) со скруглением углов `rounded-2xl` или `rounded-3xl`.
* **Интерактивность и Эффекты:** Все кнопки, вкладки и карточки должны иметь плавные микро-анимации при наведении (`transition-all duration-300 ease-in-out`). Для карточек добавь эффект приподнимания или легкого изменения прозрачности бордера (`hover:border-indigo-500/50`, `hover:-translate-y-0.5`).

## 4. Требования к SEO и Локализации

1. **Динамические метаданные (`generateMetadata`):** Функция должна генерировать уникальные, продающие заголовки и описания для каждой страницы и локали с естественным вхождением городов (Алматы, Астана, Шымкент) на соответствующем языке.
2. **Теги Hreflang (`alternates`):** В метаданных жестко прописать ссылки на языковые версии страниц (`ru-KZ`, `kk-KZ`, `en-US`), чтобы избежать дублирования контента поисковыми роботами.
3. **Структурированные данные JSON-LD (Schema.org):**
* На Главной странице внедрить скрипт `application/ld+json` с типом `EducationalOrganization`, указав страну `KZ` и обслуживаемые города (`areaServed`).
* На странице Тарифов внедрить JSON-LD с типом `PriceSpecification` для каждого тарифного плана с указанием валюты `KZT`.


4. **Семантическая HTML-верстка:** Использовать только один тег `<h1>` на страницу, секции разделять через `<section>`, подзаголовки организовывать строго по иерархии (`<h2>`, `<h3>`).

## 5. Техническое SEO и управление индексацией

1. **Файл `app/robots.ts`:** Реализуй программное управление индексацией с помощью встроенного в Next.js Metadata API для `robots`. Файл должен динамически формировать правила `Robots` для поисковых систем:
* **Разрешить (Allow):** Индексацию всех публичных страниц, включая локали: `/`, `/*/pricing`, `/*/features`, `/*/contacts`, а также динамические пути публичных курсов.
* **Запретить (Disallow):** Полностью закрыть от индексации приватные рабочие интерфейсы, страницы оплат и служебные роуты: `/*/admin/*`, `/*/dashboard/*`, `/api/*`, `/*/checkout/*/success`, `/*/checkout/*/cancel`.
* **Sitemap:** Автоматически добавлять ссылку на карту сайта (указать `https://edutech.kz/sitemap.xml`).


2. **Файл `app/sitemap.ts`:** Настрой динамическую генерацию XML-карты сайта. Ссылка на карту должна возвращать актуальный список публичных страниц и динамических курсов (запрашивая активные ID из бэкенда).

## 6. Технические ограничения и качество кода

* **Компиляция и Типы:** Код должен быть строго типизирован. Параметры страниц (динамические `params` для `landing`) должны обрабатываться с учетом асинхронности Next.js 16.
* **Стилизация:** Использовать исключительно синтаксис Tailwind CSS v4. Никаких сторонних CSS-модулей или инлайн-стилей.
* **Производительность:** Все изображения должны быть оптимизированы через `next/image` с обязательным указанием размеров и корректными `alt`-текстами.
* **Автономная проверка:** Перед отправкой Pull Request агент обязан запустить сборку проекта командой `npm run build` и исправить любые возникшие ошибки компиляции или линтера.
